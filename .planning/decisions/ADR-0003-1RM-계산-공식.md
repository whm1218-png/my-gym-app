# ADR-0003: 1RM 계산 공식 (Epley)

## 상태
채택됨 (Accepted)

## 배경
- 앱의 핵심 가치 중 하나는 기록 직후 사용자가 근력 변화를 수치로 이해하게 하는 것이다.
- 타깃 사용자는 입문자·중급자로, 복잡한 지표보다 직관적이고 널리 알려진 계산 기준이 필요하다.
- 사용자 합의 사항으로 Epley 공식을 기본 공식으로 사용하기로 결정했다.

## 대안
1. Epley
2. Brzycki
3. 복수 공식 제공(사용자 선택)

## 결정
- 기본 1RM(정확히는 e1RM) 계산 공식은 Epley로 고정한다.
- 세트별 e1RM을 계산한 뒤, 운동 단위 대표값은 해당 세션 내 최댓값을 사용한다.

## 이유
- Epley는 대중적으로 인지도가 높아 사용자 설명과 커뮤니케이션 비용이 낮다.
- 계산식이 단순해 실시간 피드백 및 테스트 작성이 쉽다.
- 초기 단계에서 공식을 하나로 고정하면 UI/설정 복잡도와 혼동을 줄일 수 있다.

## 결과/영향
- 계산식 표준: `e1RM = weight × (1 + reps/30)`.
- 보고/시각화/기록 비교는 동일 공식 기반으로 일관성 있게 제공한다.
- 추후 고급 사용자 요구가 커지면 다중 공식 지원을 별도 설정 기능으로 확장 가능하며, 해당 시점에 ADR 개정한다.
