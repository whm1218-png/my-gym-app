# ADR-0001: 모바일 프레임워크 선택 (Flutter)

## 상태
채택됨 (Accepted)

## 배경
- 본 프로젝트는 학기 일정(12주 중간 발표, 15주 최종 발표) 내에 동작 가능한 모바일 앱 데모 완성이 필요하다.
- 타깃 사용자는 운동 기록과 성장을 중시하는 입문자·중급자이며, 핵심은 빠른 입력 UX와 안정적인 로컬 기록이다.
- 팀은 초기 단계에서 서버 없이 로컬 중심 개발을 진행하기로 했고, 구현 복잡도를 낮추는 것이 중요하다.

## 대안
1. Flutter
2. React Native
3. 네이티브(iOS Swift + Android Kotlin) 병행 개발

## 결정
Flutter를 기본 프레임워크로 채택한다.

## 이유
- iOS/Android 단일 코드베이스로 개발하여 학기 일정 내 구현·테스트·데모 준비 부담을 줄일 수 있다.
- Material 3 기반 Theme 체계로 다크 모드 중심 UI를 일관되게 적용하기 쉽다.
- 로컬 저장(특히 SQLite/drift, Hive) 레퍼런스와 생태계가 충분해 빠르게 안정적인 데이터 계층을 만들 수 있다.
- 향후 서버 동기화를 붙일 때도 현재 구조를 크게 훼손하지 않고 확장 가능하다.

## 결과/영향
- 프로젝트 구조, 상태관리, 라우팅 선택은 Flutter 중심으로 표준화한다.
- 플랫폼별 네이티브 기능 의존은 최소화하고, 필요 시 플러그인 또는 플랫폼 채널로 보완한다.
- 팀 학습 곡선은 Dart/Flutter에 집중한다.
